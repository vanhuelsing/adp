===== HEADER EXPLANATIONS =====

Authorization:
  Bearer token for API authentication. Format: adp_<version>_<key>_<checksum>
  In real usage: Replace with your actual API key

X-ADP-Version:
  Protocol version. Must match the version in the request body.

X-ADP-Idempotency-Key:
  UUID v4 for idempotency. Same key + same API Key = cached response (24h)
  Prevents duplicate submissions if a request is retried

X-ADP-Signature (optional for Request Signing):
  HMAC-SHA256 signature for request integrity verification
  Format: hmac-sha256=<base64-encoded-signature>
  Computed from: <timestamp>.<method>.<path>.<content-hash>

X-ADP-Signature-Timestamp (optional for Request Signing):
  Unix timestamp (seconds since epoch) used in the signature computation
  Note: X-ADP-Signature-Timestamp uses Unix timestamp (exception to ISO 8601 rule)
  Reason: Standard in cryptographic implementations

===== SECTION 4.1 OF AUTH.MD REFERENCE =====

See auth.md Section 4.1 for Idempotency Key details:
  "Scope: Idempotency is unique within the tuple (API Key, HTTP Method, Endpoint Path)"

This ensures security: Different API Keys can use the same Idempotency-Key UUID 
without interfering with each other's deduplication.
